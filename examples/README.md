## üéØ Generated Nodes

### Rembg Background Removal Node

**Source**: [danielgatis/rembg](https://github.com/danielgatis/rembg)

A professional-quality ComfyUI node for AI-powered background removal supporting multiple models:

- **u2net** - General purpose background removal
- **birefnet-general** - High accuracy general model
- **isnet-anime** - Optimized for anime/cartoon images
- **u2net_human_seg** - Human segmentation
- **u2net_cloth_seg** - Clothing segmentation

**Features**:
- Multiple AI model support
- Alpha matting for edge refinement
- Mask output for compositing
- Session caching for performance
- Error handling with fallbacks

**Installation**:
```bash
# Install dependencies
pip install rembg>=2.0.50

# Copy to ComfyUI
cp nodes/rembg_background_removal_node.py /path/to/ComfyUI/custom_nodes/

# Restart ComfyUI
```

## üîÑ Example Workflows

### Background Removal Workflow

**File**: `workflows/background_removal_workflow.json`

A complete workflow demonstrating:

1. **Load Image** - Load source image
2. **Remove Background** - Use Rembg node to remove background
3. **Save Result** - Save the background-removed image
4. **Load New Background** - Load replacement background
5. **Composite** - Combine subject with new background
6. **Save Final** - Save the final composited image

**Usage**:
1. Import the workflow into ComfyUI
2. Load your source image
3. Choose a background removal model
4. Load a new background image
5. Execute the workflow

## üöÄ Generating More Examples

You can generate additional nodes using the framework:

```python
from src.comfyui_mcp_server_v2 import ComfyUIMCPServer

server = ComfyUIMCPServer()

# Generate from a research paper
result = server.generate_node(
    source="https://arxiv.org/abs/2301.12345",
    quality_level="production",
    focus_areas="image processing, neural networks"
)

# Generate from a GitHub repository
result = server.generate_node(
    source="https://github.com/user/awesome-project",
    quality_level="production",
    focus_areas="computer vision, preprocessing"
)
```

## üìù Node Quality Levels

The framework supports different quality levels:

- **Draft** (0.3-0.5): Quick prototypes, basic functionality
- **Development** (0.5-0.7): Working nodes with good features
